Задача в отдел - это смарт процесс для выполнения какого-то раздела проектной/рабочей документации.
Лимиты - это денежные средства, которые выделяются на эту задачу.
Перед тем как создавать задачу в отдел, весь проект рассчитывается, делится на такие задачи по которым распределяются лимиты.
Задачи делятся на подзадачи. Это происходит всегда. Даже есть подзадача всего одна (т.е. фактически является задачей), цепочка остается:
Задача > Подзадача.

Сначала заводится Задача в отдел (ЗВО). Некоторые материалы и данные тянутся из привязанной сделки.
Потом обсуждаются исходные данные и лимиты. 
На стадии выполнения заводятся подзадачи в отдел (ПВО).
ПВО тянет лимиты из ЗВО.
По итогу выполнения ЗВО назначается аккредитация - это распределение лимитов по сотрудникам согласно их участию при выполнении ПВО.
Через бизнес процессы не даем заподить подзадачи, если лимитов не хватает. Не даем назначать на подзадачи лимитов больше, чем в задаче (суммируем все лимиты всех созданных подзадач и считаем остатки каждый раз, когда пытаемся завести новую ПЗО)
