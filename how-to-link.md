# Как привязать приложение к порталу Битрикс24

## Что такое client_id и client_secret

Битрикс24 использует OAuth-авторизацию. Когда пользователь открывает приложение, Б24 выдаёт временный код, который приложение обменивает на `access_token` для REST-запросов. Для этого обмена нужны:

| Параметр | Назначение |
|---|---|
| `client_id` | Идентификатор приложения — говорит Б24, *какое* приложение делает запрос |
| `client_secret` | Секретный ключ — доказывает, что запрос делает *настоящий* владелец приложения |

Хранятся в `/var/www/b24/<slug>/env.php`. **Никогда не коммитить в git.**

---

## Порядок действий

### 1. Задеплоить приложение

```bash
bash deploy.sh
```

### 2. Открыть init.php и исправить APP_URL

Открыть в браузере:
```
https://b24.blackboxbegin.space/<slug>/init.php
```

Init.php создаст `env.php`, но запишет `APP_URL` без `/<slug>` (особенность Caddy `uri strip_prefix`).
Вручную исправить в `/var/www/b24/<slug>/env.php`:

```php
// Было:
define('APP_URL', 'https://b24.blackboxbegin.space');

// Должно быть:
define('APP_URL', 'https://b24.blackboxbegin.space/<slug>');
```

### 3. Зарегистрировать приложение в Б24

В портале Битрикс24: **Разработчикам → Приложения → Добавить приложение**

- Тип: **Серверное**
- **Путь обработчика:** `https://b24.blackboxbegin.space/<slug>/index.php`
- **Путь установки:** тот же URL (с `/index.php` на конце — обязательно)
- Права (минимум): `crm`, `user`, `placement`

После сохранения Б24 покажет `client_id` и `client_secret`.

### 4. Вписать credentials в env.php

```php
define('B24_CLIENT_ID', 'app.xxxxxxxxx.xxxxxxxx');    // из карточки Б24
define('B24_CLIENT_SECRET', 'xxxxxxxxxxxxxxxxxxxxxxxx');  // из карточки Б24
```

### 5. Установить приложение

Открыть в браузере страницу приложения в Б24 или перейти по `index.php` напрямую.
При успешной установке OAuth-токены сохранятся в `data/b24-tokens.php`.

## Несколько порталов

Этот шаблон — **single-tenant**: один экземпляр = один портал.

Для каждого дополнительного портала деплоить отдельный экземпляр под новым slug:

```
/var/www/b24/app-portal1/   ← свой env.php с credentials портала 1
/var/www/b24/app-portal2/   ← свой env.php с credentials портала 2
```

Повторить шаги 1–5 для каждого портала.

---

## Занятые порты и домены

| Что | Адрес |
|---|---|
| Все Б24-приложения | `b24.blackboxbegin.space/<slug>/` |
| dev-portal | `vibe.blackboxbegin.space` |

Б24-приложения работают через PHP-FPM, портов не занимают.
